Experimental: Thai LegalCraft

Transforming Legal Complexity into Seamless Automation

Overview

Thai LegalCraft is a Proof of Concept (POC) platform designed to democratize access to legal documentation. By bridging the gap between complex legal jargon and user-friendly interfaces, the project automates the generation of legally binding loan agreements. Built in just 3 days, this project demonstrates the power of an AI-Native Stack in rapid product development and process automation.

Check Thai LegalCraft POC Website

The Vision

Legal documents often act as a barrier for the general public due to intimidating language and rigid formats. My goal was twofold:

  • Simplify Legal Access Create a solution where users provide simple inputs and receive a professionally formatted, print-ready PDF.
  • Tech-Stack Exploration Validate the efficiency of an AI-driven development workflow using Lovable and n8n to achieve a "Fast-to-Market" prototype.
Tech Stack
  • Frontend: Lovable (AI-Native IDE) for a responsive, logic-heavy React UI.
  • Backend Automation: n8n (Low-code Workflow Engine) for document orchestration.
  • Document Engine: Google Docs API for dynamic template mapping.
  • Infrastructure: Sevalla for Static Site Hosting and Environment Variable management.
  • Collaboration: GitHub for CI/CD and version control.

I. The 3-Day Sprint: Development Journey

  • Day 1: Logic & Architecture

    I defined the core loan agreement logic, focusing on Thai-specific legal requirements. I utilized Lovable to generate a High-Fidelity Prototype that surpassed traditional wireframes. It didn't just look like a product, it functioned like one, handling real-time data binding between the form and a live A4 preview.

  • Day 2: Integration & Automation

    I shifted focus to the backend, building a robust n8n workflow. The challenge was ensuring seamless data transfer between the frontend and the Google Docs engine. Leveraging my background in DevOps, I configured the API Webhooks and handled Authentication, ensuring each node in the workflow successfully transformed raw JSON into a structured document.

  • Day 3: Refinement & Deployment

    The final push involved handling Binary Data, sending the generated PDF directly back to the user's browser for download. I addressed edge cases like Thai Baht text conversion (handling Satangs) and implemented "Smart Underscores" for empty fields to allow for manual handwriting post-printing. Finally, I deployed the frontend on Sevalla with full production environment variables.

II. Challenges & Key Learnings

  • AI as a Co-Pilot

    Using Lovable shifted my role from a pure "Coder" to a Product Manager/Business Analyst. I focused on briefing requirements and refining outcomes. The AI handled the heavy lifting of UI logic such as Thai date formatting and complex if-else display cases, allowing me to focus on the system architecture and user flow.

  • Pragmatic Problem Solving

    One of the most satisfying moments was successfully passing the PDF as binary data back to the UI. However, the true learning came from the "V1.0 Mindset." I realized that even with AI, perfection isn't the goal for a launch Timeline and core functionality are. I prioritized a working POC over a perfect UI when resource limits were met.

  • The Scalability of Low-Code

    Testing n8n allowed me to see its immense value for UX Workflow Testing and Idea Validation. While I believe for this project, a custom HTML-to-PDF engine might be more scalable for high-volume, multi-template systems, n8n is an unbeatable tool for proving an idea's worth without a massive upfront investment in time and budget.

III. Future Roadmap

  • UI/UX Polish: Adding loading overlays, full mobile responsiveness, and a more refined success state feedback loop.
  • Template Expansion: Adding other contract types, such as Rental Agreement ans Vehicle Sale Agreements.
  • User Dashboard: Implementing a "My Documents" section for users to manage, edit, and track their contract history.

Reflections

This project was more than just a coding exercise, it was an exploration of how modern tools can reduce the gap between an idea and a production-ready application. In just 72 hours, I went from a blank slate to a deployed platform, proving that the right tech stack, combined with a clear vision, can solve real-world problems with unprecedented speed.

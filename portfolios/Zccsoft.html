Case Study 1: Ziios Platform

The Challenge: Tackling Technical Debt

The Dx1 DMS platform, a Software as a Service solution for Powersports and Motorcycle Dealerships, had grown over a decade. It featured cover sales, inventory, accounting, and marketing but the front-end was burdened with serious Technical Debt.

This didn’t cause immediate user errors, but it slowed development and created UX inconsistencies.

  • UI Inconsistency: No Design System meant multiple color shades, inconsistent fonts, and duplicate code for similar components (e.g., 2-3 different nav bar code versions for the same visual component). This led to fragmented user experience.
  • CSS Fragility: Deep nesting, mixed selector practices, and numerous inline styles made debugging painful. A small change often broke unrelated parts, slowing down the team and increasing QA time.
  • Development Bottleneck: Developers spent too much time understanding and fixing old codes, slowing feature delivery and causing complaints about page speed.
The Solution: Building a Scalable Future

My approach had two goals:

  • Structural Sustainability (better code and design)
  • Strategic Business Value (new features and compliance)

I led these changes, bridging business needs and technical execution.

I. Design System & Structure: Creating a Single Source of Truth

Goal: Remove inconsistency and reduce Technical Debt.

  • Design (Figma): Built a Design System for all visual language (setting clear Color Palettes, Typography Hierarchy, and reusable components).
  • Code (SASS & Structure): Converted designs into code using SASS variables and set HTML/CSS naming standards. This cut CSS duplication and conflicts, creating a solid foundation.

II. Web Accessibility & Responsive: Expanding Reach & Compliance

Goal: Meet U.S. legal standards and support non-desktop use.

  • Accessibility (A11Y): Achieved WCAG Level AA compliance to reduce litigation risk. Integrated ARIA attributes and semantic HTML into complex UI, despite legacy 3rd-Party libraries limitations.
  • Responsive Design: Made the platform mobile-friendly. For complex tables due to 3rd Party library limitations, based on analytics showing mobile was used mainly for quick lookups. I negotiated a trade-off: instead of full mobile tables, we built a Card View with Quick Actions, accepting “good enough" experience for non-core mobile use cases, thus saving significant development time.

III. Adding High-Impact Features

Goal: Solve key pain points and create network value.

Dealer-to-Dealer Inventory Search (B2B Network)
  • Strategic Value: Turned the platform into a dealer ecosystem, helping sell slow-moving inventory and meet customer demand even when out of stock.
  • Trust & Consent: Added privacy settings allowing Dealers to control exactly what they share (unit level, catalog level, and search radius) to build trust and encourage network participation.
  • Data Integrity: Ensured data integrity by limiting searches to official manufacturer units only, guaranteeing that the core model identifier (key) was consistent across all participating dealers.
Physical Inventory for Mobile
  • The Problem: Dealers staff struggled with inconvenience to use large devices (laptops/desktops) for Physical Inventory Counting inside storage areas/warehouses.
  • The Solution: I designed a Mobile-First UI/Flow specifically for inventory counting, transforming the user's phone into a quick, specialized data entry tool. This flow was intentionally distinct from the complex desktop version, focusing solely on fast numerical input and touch-friendly, one of the most-used mobile features.

Key Takeaways

My work positioned me as a Design-Code Bridge, driving both systemic improvements and user-focused features.

Impact Highlights:

  • Structural Sustainability: Reduced bugs and speed up development by introducing design/code standards.
  • Risk & Compliance: Delivered WCAG AA compliance to avoid legal issues.
  • Strategic Decisions: Balanced business needs and technical limits (e.g., responsive tables).
  • User Impact: Created features like Mobile Inventory that solved real-world problems, proving the value of deep user context understanding.
